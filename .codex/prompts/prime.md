# 预热（Prime）：加载当前项目上下文

## 目标

通过分析项目结构、关键文档、工程文件和大文件现状，快速建立对当前代码库的整体理解，并为后续拆分或开发任务做准备。

## 流程

### 1. 分析项目结构

优先查看项目根目录和主要业务目录，重点关注：

- `Mike11/`
- `Mike21/`
- `Mike Flood/`
- `CatchMent/`
- `Common/`
- `Const_Global/`
- `.codex/prompts/`

### 2. 阅读核心文档

优先阅读以下文件：

- `README.md`
- `AGENTS.md`
- `项目介绍与源码拆分计划.md`
- `bjd_model.csproj`

### 3. 识别关键源码文件

根据当前项目结构，优先识别并阅读以下大文件或核心文件：

- `Mike11/Res11.cs`
- `Mike11/Nwk11.cs`
- `Mike11/Reach.cs`
- `HydroModel.cs`
- `Model_Ser.ashx.cs`

如果 `Res11` 已拆分，则继续查看：

- `Mike11/Res11/` 目录下各 `Res11.*.cs` 文件

### 4. 了解当前项目的技术边界

重点确认：

- 项目目标框架：`.NET Framework 4.6.1`
- 是否依赖 DHI MIKE 系列 DLL
- 是否依赖数据库访问组件
- 是否存在当前环境无法完成编译验证的外部依赖

### 5. 输出预热摘要

输出时至少覆盖以下内容：

#### 项目概述

- 项目用途
- 主要业务能力
- 主要模型引擎

#### 技术栈

- ASP.NET / .NET Framework 版本
- 主要外部依赖
- 工程组织方式

#### 当前维护重点

- 当前正在治理的大文件
- `Res11` 当前拆分状态
- 是否存在明显的结构风险

#### 后续建议

- 如果下一步是拆分，建议从哪一步开始
- 如果下一步是开发，建议优先看哪些文件

## 输出要求

- 使用中文
- 结构清晰
- 以项目实际结构为准，不套用其他项目模板
- 不要假设当前目录一定是 git 仓库；如果不是，则跳过 git 状态检查
